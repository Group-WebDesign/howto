Lesson3:

Observers


Observers is the reason why EmberJS could make the data binding possible. It watches for changes in the code, and directly fire as soon as the properties that it observes changes. That's why observers is called synchronous due to this behavior. However, observer is not the only method that Ember uses when it respond to a change. There is another method called property which works similarly. Thus, it is a good thing to be able to distinguish the difference between property and observer too in this page.

Take a look at the picture above, notice that two different methods being applied at once, which are computed a.k.a (property) and observer. As it shown in the picture, fullName function is defined by property and fullNameChanged is defined by observer. In the first method (property), when user changes either the firstName or lastName, fullName property will undergo an update following the template where it bounds. Therefore, this property is simply only provide updated value for what it watches, not anything else. On the other hand, in observer method, when user changes either firstName or lastName, the observer will see this changes and perform an action. Sort of to say that it will fire whatever codes written in the body of the method, instead of just watching the changes like property does. In this example, it will fire the person.set *line 23

For first time learning, it may seems confusing about observer method because it is really similar to property. However, to simplify things, just remember this simple conclusion. Use observer when you only want to manipulate things, but use property instead when you need to use a return value later on since only property can return something.
