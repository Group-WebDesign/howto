Lesson 2: Binding

Ember.js can bind variables together, meaning that when one variable is changed the other is changed as well. This can be incredibly useful for live updating to the user. Ember.js is unique in that it allows you to bind variables across objects too. Below is some example code for binding:

In this example the variables householdIncome for the objects husband and wife are bound using line 6 "householdIncome: Ember.computed.alias('wife.houseHoldIncome')". So when the husband or the wife gets a raise the household income for both of them is adjusted to the new value.

Another binding feature for Ember.js is One-Way-Binding. This allows a variable to be bound to another variable but not vice-versa. So, for example, changing variable foo will change variable bar, but changing variable bar will not change variable foo. Below is example code for One-Way-Binding:

In this example the variable userName in the object UserView is bound to variable fullName in the object user. The binding is one way because of line 6 "userName: Ember.computed.oneWay('user.fullName')". So when the fullName of user is set to Krang Gates the userName is also set to Krang Gates, but when userName is set to Truckasaurus Gates fullName is not changed and remains Krang Gates.
