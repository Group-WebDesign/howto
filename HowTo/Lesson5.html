Lesson 5:

Controller

Controllers in EmberJS allows the user to decorate his/ her models while providing a user interaction, without having a need to save the application to the server. Therefore, the use of controller are to decorate a model returned by the route and to list to an action/s performed by the user.


Take a look at the snippet of code above, whatever happened in these lines of code is all just a display concern (which is to display/hide a button). The controller gives a separation between logic related for user data and logic related for user display. In other words, user can test the display interface by all means without have to worry that his/her other code for data will be messed up.


The diagram above can tell EmberJS allow user to modify the user interface without disturbing the other data code. It's because of this one way line (direct chain of command) from template to model. Notice that templates will always be linked to controllers not models. With this behavior, it gives an advantage to separate display properties with model properties. Thus, logic related data and logic related display will never interact with each other and create a clean easy setup for user to code.
